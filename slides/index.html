Putting it Together

  • SSG (Static Site Generator)

SSGs

  • On my machine, an app server
  • I run it, I edit files, I see updates
  • When done, export to flat files

SSGs

  • Many options (like seriously, many)
  • You will love some, hate some, etc
  • My history: Harp.js (dead), Hugo (fast but ugh), Jekyll (woot but ruby...), 11ty (perfect for now)

Host

  • You have files, that's it.
  • Amazon S3, Azure Static Website Hosting
  • Netlify and Vercel (previous Zeit)

Here's why...

  • Faster
  • More secure
  • Shifted complexity
for shifted com, talk about wordpress+php+mysql then talk about your local dev

Here's why not...

  • Editing experience for non-devs
  • No app server may mean multiple "remote" services
for second bullet, mention email as a service

My take:

  • Jamstack "feels" more appropriate
  • Rise in client-side features
  • Rise in APIs
  • I love the complexity on my machine and only on my machine
